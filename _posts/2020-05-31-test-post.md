##### CORS-Policy 관련 에러 발생이유

SOP때문.

##### SOP란?

Same Origin Policy - 동일 출처 정책

###### 대충 정리하자면 서로 다른 출처의 자원에 접근하려 했기때문.

###### 즉 남의 영역을 멋대로 접근하려다가 보안정책에 걸림.



##### CORS란?

이런경우 cross-origin HTTP요청을 실행하여 엑세스 권한을 부여하도록 하눈 매커니즘을 가르키는말



### 해결: 

크롬 바로가기 - 속성 - 대상에 아래 문구 추가.

```
 --disable-web-security
```





##### 번외오류

```
Access to script at 'file://code.jquery.com/jquery-2.2.4.min.js' from origin 'null' has been blocked by CORS policy: Cross origin requests are only supported for protocol schemes: http, data, chrome, chrome-extension, https.
```



이건 내가 로컬에서 작업중 CORS정책에 걸린 상황.

같은 로컬상에서 작업을 해서 SOP에 위배되진 않았을거라 판단.

그러나 오류내용에 나와있듯이 origin(출처)가 null로 넘어와서 CORS에 위배.



```
브라우저는 (브라우저 별로 약간은 상이하지만) 웹에서 로컬 파일에 접근하지 못하도록 하기 위해 이런 식의 방법을 사용한다고 합니다.

즉 c:/경로/index.html에서 ajax로 c:/경로/js/module.js에 리소스를 요청한 건 동일 경로의 리소스를 요청한 것이 아니고 c:/경로/index.html에서 null/js/module.js로 리소스를 요청한 것이 되어 CORS에러가 발생한 것입니다.
```
